git使用笔记

一、介绍

Git 最初是由 Linus Torvalds 为了管理 Linux 内核开发而创建的。

Git 是一种开源的分布式的版本控制系统
每个开发者都有一个完整的代码库。
无需联网，备份还原，多人协作，分支切换

Git 是一套内容寻址文件系统

三、常用命令

git config --global user.name "xxx"
github 账号名字

git config --global user.email "xxxx"
github 账号邮箱

git init
初始化本地仓库

git add xxx
添加文件到暂存区

git commit -m "xxx"
添加文件到本地仓库，并提交注释message

git status
工作区已修改但未暂存、暂存但未提交的文件

git diff xxx
工作区与暂存区文件的差别

git log
本地仓库历史提交命令和记录

git log –pretty=oneline
只显示id和提交注释

git reset --hard HEAD^
撤销仓库最近一次提交，回到上次提交的内容，HEAD^^是上上次，’’HEAD~3是上上上次

git reflog
列出所有仓库提交命令记录(提交，合并，重置)

git reset --hard xxxxx
所有内容回到指定版本

git checkout -- xxx
指定文件回到暂存区或仓库的状态

ssh-keygen -t rsa –C “xxx”
ssh产生id_rsa 私匙 ，id_rsa.pub 公匙 ，加密算法type 是rsa，注释comment
将公匙复制，上传到github=>setting=>ssh

git remote add origin xxx
添加远程仓库
ssh连接 git@github.com:用户名/仓库名
http连接 https://www.github.com/用户名/仓库名

git push origin master
第一次加-u 全拼 –set-upstream 将 master与远程仓库连接

git clone xxx
克隆远程仓库到本地

git branch dev
创建dev分支

git checkout dev
-b 创建dev分支并转到dev分支

git branch
列出所有分支

git merge dev
将dev分支和当前分支合并

git branch –d dev
删除dev分支

git stash
将当前分支内容转到隐藏区

git stash list
列出隐藏区的分支内容

git stash apply xxx
恢复隐藏区分支内容

git stash drop
删除隐藏分支内容

git stash pop
恢复并删除隐藏区内容

git remote
查看远程库，-v 详细信息

git push origin master
将本地仓库推送远程仓库

git checkout –b dev origin/dev
创建dev分支并将远程dev分支 克隆到本地dev分支

git branch --set-upstream origin/dev dev
将本地dev分支与 远程dev分支关联起来
